# Jenkins CI/CD – Slack & Email Notification Integration (Step-by-Step)

## 🎯 Objective
Configure Jenkins to send:
- **Slack notifications** (start, success, failure).
- **Email notifications** for CI/CD builds.

---

## 🛠 Tools Required
- Jenkins (Automation Server)
- Slack (Team Communication)
- Email (SMTP service, e.g., Gmail)
- Slack Notification Plugin
- Email Extension Plugin

---

## 🚀 Step-by-Step Setup

### Step 1: Install Plugins
1. Open Jenkins → **Manage Jenkins → Plugins → Available**.
2. Install:
   - **Slack Notification Plugin**
   - **Email Extension Plugin**

---

### Step 2: Configure Slack
1. Log in to Slack and join your workspace (e.g., `VVCE`).
2. Create a public channel: `#devops-group-channel`.
3. Go to [Slack Jenkins CI Integration](https://my.slack.com/services/new/jenkins-ci).
4. Select your channel → Click **Add Jenkins CI Integration**.
5. Copy the **Credential ID** provided.
6. In Jenkins:
   - Go to **Manage Jenkins → Configure System → Slack**.
   - Workspace name = Slack workspace name (from URL, e.g., `vvce`).
   - Add credentials → Global → Secret Text → Paste Credential ID.
   - Save configuration.
   - Channel: `#devops-group-channel`.

---

### Step 3: Configure Email (SMTP)
1. Go to **Manage Jenkins → Configure System → E-mail Notification**.
2. Add:
   - SMTP Server (e.g., `smtp.gmail.com`)
   - Port: `465` or `587`
   - Credentials (email + app password)
3. Test email by sending a sample.

---

### Step 4: Create Jenkins Pipeline (Jenkinsfile)
Add Slack + Email notifications to your pipeline:

```groovy
pipeline {
    agent any

    stages {
        stage('Build') {
            steps {
                bat 'echo Building project...'
            }
        }
    }

    post {
        success {
            slackSend(channel: '#devops-group-channel',
                      message: "✅ Build SUCCESS: ${env.JOB_NAME} #${env.BUILD_NUMBER}")
            emailext(
                subject: "Build SUCCESS - ${env.JOB_NAME} #${env.BUILD_NUMBER}",
                body: "The build completed successfully.\nCheck Jenkins: ${env.BUILD_URL}",
                to: "team@example.com"
            )
        }
        failure {
            slackSend(channel: '#devops-group-channel',
                      message: "❌ Build FAILED: ${env.JOB_NAME} #${env.BUILD_NUMBER}")
            emailext(
                subject: "Build FAILED - ${env.JOB_NAME} #${env.BUILD_NUMBER}",
                body: "The build failed.\nCheck Jenkins logs: ${env.BUILD_URL}",
                to: "team@example.com"
            )
        }
    }
}
```

---

### Step 5: Verify Notifications
1. Trigger a build in Jenkins.
2. Check:
   - Slack channel → Build status message.
   - Email inbox → Notification received.

---

## 🩹 Common Issues & Fixes
| Issue | Solution |
|-------|----------|
| Slack message not sent | Verify credential ID & channel |
| Email not received | Check SMTP settings & credentials |
| Jenkins pipeline fails | Check console output for errors |

---

## ✅ Advantages
- Real-time build status updates.
- Team-wide visibility via Slack.
- Reliable email alerts for CI/CD failures.
- Faster debugging and collaboration.

---

## 🎉 Conclusion
You now have Jenkins integrated with:
- **Slack** for instant notifications.
- **Email** for CI/CD build alerts.

This ensures your team stays updated on every build and deployment automatically.
