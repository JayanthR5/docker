# Jenkins CI/CD – Slack & Email Notification Integration

## 🎯 Objective
Enable Jenkins to:
- Send **Slack notifications** for build status (start, success, failure).
- Send **Email notifications** for CI/CD pipeline builds.
- Keep the DevOps team updated automatically.

---

## 🛠 Tools Required
| Tool            | Purpose                  |
|-----------------|--------------------------|
| Jenkins         | Automation Server        |
| Slack           | Team Communication       |
| Email (SMTP)    | Build Notifications      |
| Slack Plugin    | Jenkins → Slack bridge   |
| Email Extension Plugin | Jenkins → Email bridge |

---

## 🔄 Workflow
```
Developer Push Code → GitHub → Jenkins Pipeline
                        ↓
                 Build & Deploy
                        ↓
          Slack Notification + Email Alert
```

---

## 🚀 Step-by-Step Setup

### Step 1: Install Plugins
1. Go to **Manage Jenkins → Plugins → Available**.  
2. Install:
   - **Slack Notification Plugin**
   - **Email Extension Plugin**

---

### Step 2: Configure Slack
1. Create a Slack workspace or join existing (e.g., `VVCE`).  
2. Create a public channel: `#devops-group-channel`.  
3. Go to [Slack Jenkins CI Integration](https://my.slack.com/services/new/jenkins-ci).  
4. Select your channel → Click **Add Jenkins CI Integration**.  
5. Copy the **Credential ID**.  
6. In Jenkins:
   - Manage Jenkins → Configure System → Slack.  
   - Workspace name = Slack workspace name (from URL, e.g., `vvce`).  
   - Add credentials → Global → Secret Text → Paste Credential ID.  
   - Save configuration.  
   - Channel: `#devops-group-channel`.

---

### Step 3: Configure Email (SMTP)
1. Go to **Manage Jenkins → Configure System → E-mail Notification**.  
2. Add:
   - SMTP Server (e.g., `smtp.gmail.com`)  
   - Port: `465` or `587`  
   - Credentials (email + app password).  
3. Test email by sending a sample.  

---

### Step 4: Jenkins Pipeline (Jenkinsfile)
Add Slack + Email notifications to your pipeline:

```groovy
pipeline {
    agent any

    environment {
        DOCKER_IMAGE = "yourdockerhubusername/flask-app"
    }

    stages {
        stage('Clone Repository') {
            steps {
                git branch: 'main',
                url: 'https://github.com/yourusername/Docker-Jenkins-App.git'
            }
        }

        stage('Build Docker Image') {
            steps {
                bat 'docker build -t %DOCKER_IMAGE% .'
            }
        }

        stage('Push Docker Image') {
            steps {
                bat 'docker push %DOCKER_IMAGE%'
            }
        }
    }

    post {
        success {
            slackSend(channel: '#devops-group-channel',
                      message: "✅ Build SUCCESS: ${env.JOB_NAME} #${env.BUILD_NUMBER}")
            emailext(
                subject: "Build SUCCESS - ${env.JOB_NAME} #${env.BUILD_NUMBER}",
                body: "The build completed successfully.\nCheck Jenkins: ${env.BUILD_URL}",
                to: "team@example.com"
            )
        }
        failure {
            slackSend(channel: '#devops-group-channel',
                      message: "❌ Build FAILED: ${env.JOB_NAME} #${env.BUILD_NUMBER}")
            emailext(
                subject: "Build FAILED - ${env.JOB_NAME} #${env.BUILD_NUMBER}",
                body: "The build failed.\nCheck Jenkins logs: ${env.BUILD_URL}",
                to: "team@example.com"
            )
        }
    }
}
```
